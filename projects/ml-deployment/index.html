The ML Deployment Pipeline
01
Phase 1 — Data
Data Preparation & Feature Engineering
Raw data is collected, cleaned, and transformed into meaningful features. This is the most time-consuming phase — poor data quality here propagates errors through every downstream step.
# Load and prepare training data import pandas as pd from sklearn.preprocessing import StandardScaler df = pd.read_csv('clinical_data.csv') # Feature engineering df['mutation_burden'] = df['total_mutations'] / df['genome_size'] df['hrd_positive'] = (df['hrd_score'] >= 42).astype(int) # Train/test split from sklearn.model_selection import train_test_split X_train, X_test, y_train, y_test = train_test_split( df[features], df['platinum_response'], test_size=0.2, random_state=42 )

02
Phase 2 — Training
Model Training & Validation
Model is trained, hyperparameters tuned via cross-validation, and performance evaluated on held-out test data. SHAP values explain which features drive predictions.
from xgboost import XGBClassifier from sklearn.model_selection import cross_val_score import shap # Train XGBoost model model = XGBClassifier( n_estimators=200, max_depth=4, learning_rate=0.05, use_label_encoder=False ) model.fit(X_train, y_train) # Cross-validation scores = cross_val_score(model, X_train, y_train, cv=5, scoring='roc_auc') print(f"CV AUC: {scores.mean():.3f} ± {scores.std():.3f}") # SHAP explainability explainer = shap.TreeExplainer(model) shap_values = explainer.shap_values(X_test)

03
Phase 3 — Serialisation
Save the Model
The trained model is serialised to disk. The model file, preprocessor, and feature schema are versioned together — never save the model alone without its preprocessing pipeline.
import joblib import json # Save model + preprocessor as a bundle model_bundle = { 'model': model, 'scaler': scaler, 'feature_names': features, 'model_version': 'v1.0.0', 'trained_on': '2026-03-09' } joblib.dump(model_bundle, 'model_bundle.pkl') # Save feature schema for validation with open('feature_schema.json', 'w') as f: json.dump({'features': features, 'dtypes': dtypes}, f)

04
Phase 4 — Serving
Build a Prediction API
A FastAPI application loads the model bundle and exposes a REST endpoint. Input validation, error handling, and SHAP explanations are built in — the API is the contract between the model and the world.
from fastapi import FastAPI from pydantic import BaseModel import joblib, shap, numpy as np app = FastAPI(title="Platinum Response Predictor") bundle = joblib.load("model_bundle.pkl") model, scaler = bundle['model'], bundle['scaler'] class PatientFeatures(BaseModel): tp53_mutated: bool brca1_mutated: bool hrd_score: float tmb: float msi_status: int @app.post("/predict") def predict(patient: PatientFeatures): X = scaler.transform([patient.dict().values()]) prob = model.predict_proba(X)[0][1] label = "Sensitive" if prob > 0.5 else "Resistant" return {"prediction": label, "confidence": round(prob, 3)}

05
Phase 5 — Containerisation
Dockerise & Deploy
The API is containerised with Docker for reproducibility, then deployed to a cloud platform. Render.com, HuggingFace Spaces, and AWS Lambda all support free-tier model serving.

06
Phase 6 — Monitoring
Monitor, Drift Detection & Retrain
Models degrade over time as data distributions shift. Logging predictions, tracking performance metrics, and detecting feature drift ensures the model stays reliable in production.
from evidently import ColumnMapping from evidently.report import Report from evidently.metric_preset import DataDriftPreset # Compare reference vs production data distributions report = Report(metrics=[DataDriftPreset()]) report.run( reference_data=reference_df, current_data=production_df ) # Trigger retraining if drift detected drift_detected = report.as_dict()['metrics'][0]['result']['dataset_drift'] if drift_detected: trigger_retraining_pipeline()

Training

Industry-standard ML frameworks for classification, regression, and ensemble models. XGBoost excels on tabular clinical data.
Explainability
SHAP
SHapley Additive exPlanations — the gold standard for explaining individual predictions and global feature importance.
Versioning

Track experiments, log metrics, register models, and manage the full model lifecycle across training runs.
Serving
FastAPI
High-performance Python API framework. Automatic OpenAPI documentation, input validation via Pydantic, async support.
Containerisation
Docker
Package model + dependencies into a reproducible container. Eliminates "works on my machine" problems in production.
Monitoring

Open-source ML monitoring — data drift detection, model performance tracking, and automated reporting.
